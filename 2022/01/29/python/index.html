PYTHON

基础知识

python解释器的作用

  • 用来将代码文件转换成计算机能识别的机器代码

注释

  • 单行注释: # 注释内容
  • 多行注释: ‘’’ 注释内容,可换行 ‘’’

变量的概念

  • 变量就是一种代号:用来代表的数据

标识符的命名规则

  • 标识符由字母、下划线和数字组成，且数字不能开头
  • 标识符定义区分大小写
  • 定义时需要见名知意
  • 类名使用大驼峰式命名法
  • 其它标识符(变量,函数名,模块名,包名) 都使用下划线风格命名方式

格式化操作符的使用

  • 格式化字符串第一种形式: 在字符串中使用格式占位符

    • %nd n 表示占符宽度,如果数字宽度小于指定宽度,左侧使用空格补齐,表示需要一个整数,
    • %s 表示需要一个字符串
    • %f 表示需要一个浮点数
    • %.nf n 表示一个数字,作用是保留小数精度,小数点后面保留几位

  • 格式化字符串第二种形式:

    • f’字符串{变量}’

数据类型转换

  • GBK
  • Unicode
  • UTF-8
  • str() 将数据转换成字符串
    chr() 将数字转换成对应的字符
    ord() 将字符转换成对应的字符编码 ASCII码

  • continue结束本次循环，进入下一次

  • for 循环变量 in range(a, b):

函数

函数的实参和形参

  • 在函数调用时,括号内参数列表中的数据都是实参(可以使用真正的数据,也可以使用变量,甚至可以是另外一个函数的结果)

组包和拆包

  • 组包: 将多个值同时赋给一个变量时,解释器会进行自动组包操作

  • 拆包: 将一个容器值(元组),里面的多个数据同时赋值多个变量,解释器会进行拆包操作

    • 注意: 拆包要注意被赋值的变量个数和元组中的值个数相同

函数返回多个值

  • 当在函数中,需要返回多个值的情况下,可以在return 后面直接将多个值写出来

    • 解释器在返回时,会将多个值自动进行组包

默认值参数

混合参数

    def func(a,b,c,d,e, *args,f=1,g=2,**kwargs):

      函数体代码

  • 不可变类型:

      数字 int
  字符串 str
  浮点数 float
  布尔类型 bool
  元组  tuple

  • 可变类型:

      列表 list
  字典 dict
  集合 set

文件的操作

以文本方式打开方式的模式

  • r -> rt -> read text 以文本方式打开文件读,文件存在,打开成功,文件不存在,打开失败
    w -> wt -> write text 以文本方式打开文件写,不管文件是否存在,都会新创建一个新文件
    a -> at -> append text 以文件方式打开文件追加,文件不存在,创建文件,文件存在,那么打开文件然后将光标移动到文件的最后

以二进制形式打开文件的模式

  • rb 以二进制形式打开文件读取
    wb 以二进制形式打开文件写入
    ab 以二进制形式打开文件追加

文件读取的几种方式

  • read() # 一次性将整个文件内容读取出来,不适合大文件读取
    read(size) # 掌握 读取指定的字节数的文件内容 ,如果文件最后没有内容了,那么read的返回值是空字符串 , “”
    readline() # 读取一行
    readlines() # 一次性将整个文件内的内容以行的形式读取出来,放到一个列表中

文件的写入

  • write(content) 该函数用来向文件中写入数据
    前提: 使用该函数时,打开文件时,必须要给定写入权限,要具有写入模式

面向对象

类: 是对现实事物的属性和行为的抽象

  • 类的名称:类名
    类的属性:一组数据
    类的方法:允许对进行操作的方法 (行为)

对象: 是类的一种实例表示

创建对象和使用对象的格式

  • 对象名 = 类名(参数列表….)

    对象调用方法的格式:
    对象名.方法名(参数列表)

  • 注意:
    方法中参数列表中的第一个参数self,不需要手动传递,这个参数是由解释器在执行程序时,自动传递的
    默认会将当前调用方法的对象引用传递进去

init 方法的基本使用和作用

  • 在python中,以前后各有两个下划线的这种方式命名的标识符,称为魔法属性,或魔法方法
    这些魔法属性或魔法方法,是python中内置的,预先定义好一些功能的属性或方法

    init 方法的作用是为对象进行初始化 initialization
    该方法会在创建对象时,自动调用

    一般会将对象所具有的属性放到这个方法中,进行初始化,不在使用动态绑定为对象添加属性

    在这个方法使用使用 self.属性名 = 值 方式添加属性

  • 格式:

      def __init__(self, name, age):
      # 函数体

str 方法

  • 格式化对象

      该方法返回一个字符串值
  当使用 str() 做类型转换时,会自动调用 该方法

    当一个自定义类,没有实现该方法时,默认打印格式是 <模块名.类名 object at 地址>
    eg:<__main__.Cat object at 0x02176610>

  • 调用该方法的场景:

      1. 通过 print() 函数打印时,会自动调用该方法
  2. 通过 str() 函数对自定义的对象进行类型转换时,会自动调用该方法

del 方法

  • 该方法是用来在销毁对象时,回收释放资源使用的方法
    该方法也是自动调用
    当在使用 del 对象时,会调用方法

类的复合（类中属性是对象）

  • 在设计一个类时,类中包含其它类的对象,称为类的复合（难点）

私有属性的定义和使用

  • 在python抽象类时,默认定义的变量是公有的
    公有的变量,可以通过对象在任何位置访问

    类中公有属性一般不建议使用,破坏程序的封装性,在类的外部可以通过对象直接访问,

    对于数据来说,相对不安全

    类中公的方法的作用是为外部提供一种接口,通过这个接口可以来实现相应的交互操作

# 私有属性:
  因为公有属性的不安全,可以将属性设置成私有,私有属性只能在类的内部使用,而在类的外部是不可以使用的


# 私有方法
  将不想公开的实现数据和逻辑通过私有方法进行封装,可以隐藏关键代码.
  如果想使用私的方法功能时,可以在类的内部使用,或者在类的外部通过类中提供的公有方法来间接使用
  • 私有属性的优点
    set/get方法对私有属性操作时的好处:
    1. 提供精确的访问控制权限
    2. 隐藏实现细节,让代码更安全
    3. 可以提供更加安全的数据精度控制

继承

  • 概述

    • 程序中的继承,遵循生活的继承方式
      被继承的类: 父类, 超类, 基类
      衍生出的类: 子类, 派生类

    新式类中:所有的类,都直接或间接继承 object 类

    object 类是所有类的父类

    在继承时,子类拥有父类的所有的内容

    继承的作用:

      复用代码

  • 格式

    • class 子类名(父类名):
      pass

    在出现继承后
    当子类对象去调用方法时,会先在子类中去查找 方法,如果有,那么就执行,如果没有找到,去到父类中去查找 该方法,如果找到执行
    找不到再到上一级类中去查找 直到找到object类,如果还没有那么就报错

  • 调用

      1. 子类使用继承父类的属性和方法
      • 在类出现继承时,子类现阶段,可以继承父类的所有的属性和方法
      1. 子类不能继承父类的私有属性和私有方法
      1. 子类中访问父类中的私有属性和方法
        • 如果想使用父类中的私有属性和私有方法,那么就需要在父类中,为这些私有的属性和方法,提供相应的公有的接口方法来间接访问

    • 如何为父类中的属性进行初始化

      • 在子类中如果定义了自己的初始化方法,那么这时父类的初始化方法就不会再执行了
        那么就导致父类中定义的属性无法初始化,子类对象就无法使用继承自父类的属性
        如果想正常初始化,需要在子类的初始化方法中,手动调用父类的初始化方法

        • 现阶段调用格式:
          父类名.init(self,父类中需要属性参数列表)
      1. 子类中重写和调用父类的方法
      • 在子类继承父类时,子类会拥有父类中的方法.
        但是子类并不需要或不满足父类方法中提供的功能,这时就需进行对父类方法重写

    重写之后,调用该方法时,执行谁?
    答案: 调用重写后子类中的方法

      1. 子类方法中调用父类的同名方法

多层继承

  • 继承关系有多层,祖孙三代

多继承

  • 格式

    • class 类名(父类名1,…..):
      pass

  • 初始化

    • 父类名调用 理解
      super()调用 使用
      super(类名，self) 了解
    • super执行过程:
      在 self 这个对象的所属类中,通过 mro 找到方法解析顺序
      在顺序中,找当前类名的下一个类来初始化或查找方法
在多继承时,如果继承的多个类同时继承同一个父类,那么这时会出现初始化问题
这个共同父类会被初始化多次.

类名.__mro__ 得到了一个元组,元组中的元素是当前类在继承关系上的一个顺序

方法解析顺序
method relation ordered

这个顺序不是我们确定的,是由在确定某个类的继承关系关系后,由解释器来确定这个顺序


super对象是基于类的mro顺序来对当前类进行初始化的
基于这个mro顺序可以保证,所有的继承关系的上类,都只执行一次


mro -> 一个类的继承书写顺序会影响mro顺序

  • 使用指定父类中方法：

    • 父类名.方法（）

多态（多种形态）

      当调用一个方法名的时候,得到的结果不同
  在一般的面向对象语言中,多态是由继承来实现的
  但是在python中,python,天生具有多态性
  • 鸭子类型： Duck Typing

实例对象属性和实例方法

  • 实例对象: self -> 通过类创建出来的对象
    通过这个实例对象调用的属性和方法,称为实例属性和实例方法

类对象和类属性

  • 类对象: 程序中定义的类,在运行程序时,解释器会先创建一个类对象,可以理解成一个模板对象, 工程机
    所有的实例对象,都是山寨类对象

  • 定义:

    • 类属性是定义在类中所有方法之外的属性(可以理解成是类中的全局变量)

  • 使用:

      1. 在实例对象属性没有和类属性重名的情况下,那么可以通过实例对象来引用类属性(但不建议这么用)
      1. 类名.类属性

  • 作用:

    • 可以通过类属性来在各对象之间进行传递数据

类方法

  • 定义:

    • @classmethod # 这个装饰器的作用是让解释器在执行类方法时,自动将类对象传给该方法
      def 类方法名(cls): # cls用来接收当前的类对象,通过这个类对象可以使用类属性
      pass

  • 调用:

    • 类名.类方法名()

  • 作用：

    • 当在封装一些类时,这个类中存在很多功能方法,但是这些方法不需要实例对象来调用时,可以使用类方法

静态方法

  • 静态方法和类方法类似,区别在于静态方法不需接收任何参数(实例方法接收实例对象,类方法接收类对象)

    静态方法就是一组普通的函数被组合到了一起.

  • 定义：

    • @staticmethod # 一旦被这个装饰器所装饰,就是告诉解释器,这个方法什么参数也不传
      def 静态方法名():
      pass

  • 使用：

    • 和类方法相同,也是使用 类名.静态方法名()

  • 作用：

    • 静态方法的作用等同类方法,但是类方法中可以通过 默认参数 cls 访问类中的类属性
      当一个类实现的功能,不需要类属性时,那就可以使用静态方法

  • 一般类方法和静态方法,都是用来实现一些工具类的时候进行定义

如何应用这些属性和方法?

  • 实例对象: 实例属性和实例方法

  • 类对象: 类属性和类方法

  • 静态方法:

    • 就是一堆普能的函数被封装到了类中.一般用这种方法来对功能进行分类

  • 语法:
      try:

else：

finally:

模块

导入模块

  • 方式一：

      import 模块名

    方式二：

        from 模块名 import 成员名

  • 别名：

      import 模块名 as 别名
  from 模块名 import 成员名 as 成员别名

  • 两种导入方式区别：
    1.import方式,是拿到了模块对象的引用地址,通过这个地址来使用模块中的成员
    2.from-import方式,相当于将被导入模块中的成员,复制了一份到本地模块中.所以可以直接使用成员来访问被导入的成员

all

  • all 是一个魔法属性,类型是一个列表,元素是字符串类型
    作用:
      可以改变在使用 from-import 方式导入模块时,成员的导入规则

name

  • 作用:
    如果__name__ 出现在当前执行的模块中时,得到的值是 main 这个字符串,用来表示执行的是当前文件
    如果__name__ 出现在在了被导入模块中时,得到的值是被导入模块的模块名

包

  • 就是一个文件夹,无论里面是否有__init__.py

导入包中的模块

  • 方式一:

      通过包和模块的路径,明确的告诉解释器,要导入哪个包里的哪个模块
  import 包名.模块名
  from 包名 import 模块名
  from 包名.模块名 import 成员名

  • 方式二:

      import 包名
  from 包名 import *

  在导入时,不会指定模块名,只指定包名
  这时,默认情况下(__init__.py 为空)哪个模块都不会导入

  为了能让模块正常导入,这里需要 __init__.py 文件中设置可以被导入的模块
  通过方式:
  from . import 模块名

  注意:
  __all__ 只对 from 包名 import * 方法生效,对 import 包名 方式无效
  __all__ 在 __init__.py 文件中的使用了解即可
